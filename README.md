## Open Source Highlights

> Public, original repositories with more than 2,000 source lines and more than 2 commits, ordered by overall impact.

### [GitNexus-v2](https://github.com/dporkka/GitNexus-v2)
GitNexus is a browser-native code intelligence engine that turns repositories or ZIP files into an interactive knowledge graph with built-in Graph RAG. It stands out as one of my most ambitious experiments in developer tooling because the full indexing and analysis flow runs client-side through WebAssembly.

### [eleventy-blog](https://github.com/dporkka/eleventy-blog)
This is the Eleventy starter behind davidporkka.com, adapted from Supermaya with custom full-screen layouts, additional common pages, and a more tailored visual system. It is one of my most visible open-source templates and a good example of how I approach content-first site building.

### [headless-wp-start-boilerplate](https://github.com/dporkka/headless-wp-start-boilerplate)
A headless WordPress frontend starter built around TanStack Start, type-safe generated fragments, and AI-agent-friendly project conventions. It is designed for teams that want a modern WordPress-backed frontend without giving up structure, resilience, or developer speed.

### [100y-saas](https://github.com/dporkka/100y-saas)
100y-saas is a production-ready multi-tenant SaaS platform built around Go, SQLite, and a deliberately boring stack meant to last for decades. It packages authentication, analytics, subscriptions, jobs, and rate limiting into a system optimized for durability over trend-chasing.

### [Cero](https://github.com/dporkka/Cero)
Cero pushes the low-maintenance idea even further with a C, SQLite, and server-rendered HTML stack that avoids unnecessary dependencies. It is a strong expression of my interest in simple systems that stay understandable, portable, and operational for a very long time.

### [go-htmx-supabase-saas](https://github.com/dporkka/go-htmx-supabase-saas)
This project is a Go, Gin, HTMX, and Supabase SaaS starter with a plugin architecture and a modern UI layer. It shows how I like to combine fast backends, lightweight interactivity, and extensibility without overcomplicating the stack.
